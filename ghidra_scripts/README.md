# Ghidra Scripts for MemTools

These scripts work with C header files generated by `sparsestruct.GenerateCDefinitions`.

## Installation

Copy the `.py` files to your Ghidra scripts directory:
- `~/.ghidra/.ghidra_<version>/scripts/` (user scripts)
- Or use `Script Manager > Manage Script Directories` to add this folder

## Scripts

### RecursiveTypeApplier.py

Simple version: Position cursor on an existing struct data definition and run. The script will:

1. Read all pointer fields in the struct
2. For each non-null struct pointer, apply the target type at that address
3. Recursively process all child structs
4. Skip already-visited addresses (cycle-safe)

### RecursiveTypeApplierInteractive.py

Enhanced version with two modes:

**From cursor mode**: Same as the simple version.

**Interactive mode**: Prompts for:
- Address (hex format, e.g., `0x12345678`)
- Type name (must exist in Data Type Manager)

Additional features:
- Handles arrays of struct pointers
- Max depth limit (100) to prevent runaway recursion

## Workflow

1. Import your C header into Ghidra: `File > Parse C Source`
2. Apply the root struct type to your starting address
3. Run `RecursiveTypeApplier.py` with cursor on that address
4. All child structs will be automatically typed

## Cycle Detection

Both scripts track visited addresses to prevent infinite loops when struct graphs contain cycles. Each address is only processed once.
